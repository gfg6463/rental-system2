# دليل رفع وتشغيل المنصة على السيرفر السحابي (Google Cloud Platform)

هذا الدليل يشرح بالتفصيل كيفية رفع وتشغيل نظام تأجير السيارات (الواجهة والخلفية) على سيرفر سحابي (VM Instance) بنظام Ubuntu Linux على منصة Google Cloud، وفتحه من أي جهاز آخر (جوال أو كمبيوتر) بشكل كامل وسلس.

---

## 1. الخطوة الأولى: إنشاء السيرفر (VM Instance) على Google Cloud

1. ادخل إلى لوحة تحكم [Google Cloud Console](https://console.cloud.google.com/).
2. انتقل إلى **Compute Engine** -> **VM Instances**.
3. اضغط على **Create Instance**.
4. قم بتهيئة الإعدادات كالتالي:
   * **Region & Zone:** اختر المنطقة الأقرب لك (مثال: `europe-west9` لباريس أو `me-central1` للدمام/السعودية إذا كانت متوفرة).
   * **Machine configuration:** الحجم الاقتصادي `e2-micro` أو `e2-medium` كافٍ جداً للتجربة والتشغيل الفعلي.
   * **Boot disk:** اضغط على Change واختر:
     * **Operating System:** `Ubuntu`
     * **Version:** `Ubuntu 22.04 LTS` (نسخة مستقرة).
   * **Firewall:** ضع علامة صح على:
     * `Allow HTTP traffic`
     * `Allow HTTPS traffic`
5. اضغط على **Create** وانتظر اكتمال تشغيل السيرفر وحصوله على **External IP** (عنوان IP الخارجي).

---

## 2. الخطوة الثانية: فتح بورتات الاتصال في جدار الحماية (Firewall Rules)
بشكل افتراضي، يحجب جوجل كلاود المنافذ الخارجية. يجب فتح المنفذين `3000` (للواجهة) و `8000` (للخلفية).

1. في شريط البحث العلوي بجوجل كلاود اكتب: **Firewall** (أو VPC firewall rules).
2. اضغط على **Create Firewall Rule** وقم بتعبئة الحقول كالتالي:
   * **Name:** `allow-rental-system`
   * **Targets:** اختر `All instances in the network`
   * **Source IPv4 ranges:** اكتب `0.0.0.0/0` (للسماح بالوصول من أي مكان).
   * **Protocols and ports:** اختر **Specified protocols and ports**، ثم ضع علامة صح على **TCP** واكتب: `3000, 8000`
3. اضغط على **Create**.

---

## 3. الخطوة الثالثة: نقل الملفات إلى السيرفر

يمكنك ضغط مجلد `rental_system_deployment` ورفعه مباشرة إلى السيرفر، أو رفع الملفات عبر بروتوكول SFTP (باستخدام برامج مثل FileZilla أو MobaXterm) بربطها بـ IP السيرفر ومفتاح SSH الخاص بك.

بعد نقل المجلد وفك ضغطه على السيرفر (مثال في المسار الرئيسي للـ User: `~/rental-system`)، اتبع خطوات التثبيت والتشغيل التالية.

---

## 4. الخطوة الرابعة: تثبيت الملحقات والتشغيل المستمر (PM2)

اتصل بالسيرفر عبر SSH من متصفح جوجل كلاود أو سطر الأوامر، ثم نفذ الأوامر التالية بالترتيب:

### أ. تحديث السيرفر وتثبيت Node.js (نسخة 20)
```bash
# تحديث الحزم
sudo apt update && sudo apt upgrade -y

# تثبيت Node.js و NPM
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### ب. تثبيت محرك التشغيل الخلفي (PM2)
PM2 هو نظام لإدارة عمليات Node.js يضمن تشغيل الواجهة والخلفية في الخلفية بشكل دائم وإعادة تشغيلهم تلقائياً في حال توقف السيرفر أو حدوث خطأ.
```bash
sudo npm install -g pm2
```

### ج. تشغيل السيرفر الخلفي (Backend)
```bash
# الانتقال لمجلد الخلفية
cd ~/rental-system/backend

# تثبيت الحزم المطلوبة للخلفية
npm install --omit=dev

# تشغيل الخلفية عبر PM2
pm2 start server.js --name "rental-backend"
```

### د. بناء وتشغيل الواجهة (Dashboard)
```bash
# الانتقال لمجلد الواجهة
cd ~/rental-system/apps/dashboard

# تثبيت حزم الواجهة
npm install --omit=dev

# بناء تطبيق Next.js للإنتاج (Production Build)
npm run build

# تشغيل الواجهة عبر PM2 على البورت 3000
pm2 start npm --name "rental-frontend" -- run start
```

---

## 5. الخطوة الخامسة: حفظ تشغيل البرامج وتفعيل التشغيل التلقائي مع السيرفر

لضمان عمل الواجهة والخلفية تلقائياً حتى لو قمت بإعادة تشغيل (Reboot) للسيرفر السحابي:
```bash
# حفظ العمليات الحالية في PM2
pm2 save

# تفعيل التشغيل التلقائي مع إقلاع النظام
pm2 startup
```
*(سيظهر لك أمر في المخرجات يبدأ بـ `sudo env PATH=...` قم بنسخه ولصقه في الترمينال لتفعيل الخدمة).*

## 🏁 طريقة الوصول للمنصة الآن:
الآن، يمكنك الدخول للمنصة وتصفحها من أي جوال أو جهاز كمبيوتر في العالم عبر كتابة IP الخارجي الخاص بالسيرفر السحابي متبوعاً بالمنفذ 3000:
* رابط الواجهة: `http://[سيرفر_الـ_IP_الخارجي]:3000`
* مثال: `http://34.120.50.90:3000`

*(سيقوم المتصفح تلقائياً بالتحويل لصفحة تسجيل الدخول، ويمكنك الدخول بحساب bader وكلمة مرور admin123، وستعمل لوحة الأسعار وجلب البيانات بالكامل).*
